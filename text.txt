---
Running everything

Option A — Development (venv, one command)

With venv activated, from D:\AI-HPS\backend:
python run_dev.py
This launches all 6 services concurrently in one terminal. Press Ctrl+C once to stop all.

To test them quickly:
# Verify infrastructure first
python check_setup.py

# Then start services
python run_dev.py

Option B — Docker (recommended, one command)

From D:\AI-HPS\docker:
# First time — builds all 6 Python service images (~5 min, downloads sentence-transformers etc.)
docker compose up --build

# After first build
docker compose up -d        # start in background
docker compose logs -f      # follow logs
docker compose down         # stop everything

Why Docker is better:
- One command instead of 6 terminals
- Services restart automatically if they crash
- Nginx routes all traffic through port 80 (no per-service port juggling)
- Matches how you'd deploy to a server
- Redis/RabbitMQ/services all on the same Docker network

The only tradeoff: first docker compose up --build takes several minutes because it installs sentence-transformers (which pulls PyTorch) inside the container. Subsequent starts are instant.
